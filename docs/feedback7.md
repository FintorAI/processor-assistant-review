That's a **flag only**: its suggestion text tells the *processor* to go find the missing bank statement/asset doc — there is no `_write_fields(...)` call anywhere in this rule, and there can't be, since there's no source document data to write from (the VOD entry exists in Encompass; it's the supporting doc that's missing). The only two Encompass *writes* in this tool are `update_vods` (fills blank subfields on an existing VOD entry from a matched doc) and `add_vods` (adds a VOD entry from a bank statement that has none) — both go doc → Encompass, the opposite direction from this fix. If the loan for this thread still showed one flag per account instead of one per bank, that run predates the fix — rerunning against current code should show the lumped version.
