# Changelog — 2026-06-04

New **1003 URLA Lender** workflow step + step renumbering. Manner-held / estate
ownership moved out of Borrower Vesting into a dedicated early step.

---

## 1. New step: STEP_03 "1003 URLA Lender" (substep 3.1, `update_urla_lender`)

Added a dedicated step for the 1003 URLA Lender "how title will be held" fields,
placed **before** 1003 URLA Page 1 (first of the URLA steps).

- `definitions/step_03_urla_lender.yaml` — new step definition.
- `output/tools/update_urla_lender.py` — new tool (`FACTORY-LOCK: true`).

What it owns:
- **Manner in Which Title Will Be Held** (field `33`) + **`URLA.X138`** — computed
  from property state, marital status, co-borrower/NBS presence, and borrower sex
  (written when empty; NV forces *As Joint Tenants* for married couples).
- **Estate Will Be Held In** (field `1066`) — auto-set to `FeeSimple` for standard
  residential loans (blank or Leasehold).
- Surfaces **Attachment Type** (`CX.ATTACHMENT.TYPE`) and **Property Type**
  (`1041`) as an info flag for the processor to verify against the listing.
  - **TODO:** automated Attachment/Property-Type-vs-listing validation
    (Google/MLS lookup) — not implemented; flagged for manual verification only.

The manner-held computation (state rules, `_MANNER_TO_URLA_X138` mapping,
`_manner_held_compatible`) was **moved** from `update_borrower_vesting.py`.

## 2. Borrower Vesting now reads field 33 (no longer writes manner/estate)

`update_borrower_vesting` (now substep **9.1**) no longer computes or writes
Manner Held (`33`/`URLA.X138`) or Estate (`1066`). It **reads** field 33 only,
and flags if it is empty (the 1003 URLA Lender step should have set it before
Build Final Vesting). Dead manner-held helpers/constants removed from the tool;
`step_09_borrower_vesting.yaml` updated (dropped the field-33 write, manner rule
is now read-only/existence-check).
